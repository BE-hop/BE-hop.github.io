# BE-hop Project Memory Index

Read this index after the root `AGENTS.md`, then open only the memory file relevant to the current task.

## Memory Files

- `memory/project.md`: stable architecture, product position, and long-lived constraints.
- `memory/decisions.md`: dated decisions with evidence and invalidation conditions.
- `memory/workflows.md`: commands and operational workflows verified in this repository.

## Update Policy

- Repository-local memory may be updated automatically after substantive work because the site owner granted permission on 2026-07-10.
- Record only durable facts, decisions, constraints, or verified workflows.
- Every entry must include a verification date or dated decision, evidence, and an invalidation condition.
- Update an existing entry when reality changes; do not append a duplicate task log.
- Never write Codex global memory without separate explicit permission.
