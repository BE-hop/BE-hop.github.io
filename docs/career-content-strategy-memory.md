# Career Content Strategy Memory

## Purpose

This file is the long-term strategy memory for aligning BEhop Experience with career trends, job-market signals, and portfolio positioning. It guides the English-facing website content while preserving traceability to source batches under `career-trend-file/`.

Use this file before changing the homepage, Works, AI products, AI gallery, blog strategy, professional positioning, or contact-oriented content.

## Update Protocol

- Every new career-trend source batch must create or update a dated folder under `career-trend-file/YYYY-MM-DD/`.
- Raw files belong in `career-trend-file/YYYY-MM-DD/sources/` and must stay local-only.
- Each batch needs an `index.md` with file names, types, hashes, source notes, analysis status, and analysis references.
- Each analyzed batch needs an `analysis.md` explaining source-level findings and website implications.
- Before analyzing new files, compare their hashes against previous indexes. Do not repeat analysis for files already marked `analyzed` unless explicitly requested.
- Long-term, reusable conclusions should be merged into this memory file.
- Public website content should be English-first unless a Chinese mirror is specifically requested.
- Communicate planning, implementation summaries, and owner-facing decisions in Chinese by default.

## Work Direction Memory

### AI-Assisted Design Product Direction

Target role language for the website:

- AI product manager for design tools
- AI-assisted design workflow builder
- Design-domain data and evaluation contributor
- AIGC creative workflow designer
- AI Agent / design productivity explorer

Important hiring signals:

- Ability to analyze user needs and pain points.
- Ability to convert ideas into prototypes, PRDs, workflows, or product features.
- Ability to work with technical teams and understand system logic.
- Ability to use AI tools, prompts, scripts, and lightweight coding to validate ideas.
- Ability to turn design judgment into measurable evaluation criteria.
- Ability to research global AI tools, Agent products, and AIGC workflows.

Keywords to preserve in future English-facing content:

- AI-assisted design workflow
- design evaluation
- visual judgment
- prototype-to-tool
- product thinking
- user research
- workflow automation
- AIGC workflow
- AI Agent
- multimodal generation
- data-informed iteration
- design quality criteria

## Personal Portfolio Direction Memory

The portfolio should show evidence of thinking, not only final images.

Prioritize projects that demonstrate:

- A real design or workflow problem.
- The owner's role and decision responsibility.
- Tools used, including Rhino, Python, AI tools, image processing, prompt workflows, or scripts.
- A clear process from brief to output.
- Evaluation logic: what counted as good, usable, clear, accurate, or efficient.
- Iteration and constraints.
- Reflections that reveal design judgment and product judgment.

RU-LineArt should be treated as a flagship case because it connects design workflow pain points, image processing, Rhino output, user scenario definition, and prototype iteration.

## Website Content Guidance

Homepage:

- Lead with a concise English identity around design, AI workflows, and product-minded making.
- Avoid generic "full-service studio" positioning.
- Surface proof points: selected projects, AI tools, experiments, and writing.

Works:

- Each case should include brief, role, constraints, tools, process, decision logic, output, and reflection.
- Use project language that helps recruiters understand transferable judgment.

AI Products:

- Present AI products as experiments or tools with clear users, problems, workflows, features, limitations, and next steps.
- Show how product decisions were made, not only what the feature does.

Blog:

- Build English-first professional visibility around AI design tools, AIGC workflows, design evaluation, AI Agent thinking, and product experiments.
- Use posts to explain methodology, not just announce updates.

Gallery:

- Treat gallery work as visual evidence of exploration. When possible, attach tool chain, prompt direction, selection criteria, and design judgment.

## Avoid / Do Not Emphasize

- Do not turn the site into a hard-selling service funnel.
- Do not add pricing packages, limited-time offers, exaggerated guarantees, or aggressive conversion copy.
- Do not make unverifiable claims about AI expertise.
- Do not present generic AI enthusiasm without project evidence.
- Do not over-index on final visuals while hiding process, judgment, or constraints.
